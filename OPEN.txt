ひらけないとき（このフォルダ english-lesson-pwa のなかにいること）

1) ターミナルをひらく

2) このフォルダに移動（Finder で english-lesson-pwa を ターミナルにドラッグ）

   cd （スペースのあとドラッグ）

3) これを 1行ずつ

   python3 serve_lan.py --open

   → ブラウザが http://127.0.0.1:8765/ をひらく

4) エラーが出たら

   • 「index.html がみつからない」→ cd まちがい（別フォルダにいる）
   • 「Address already in use」→ python3 serve_lan.py 8877 --open

5) iPad から LAN でつなぐ（まずここまで試す）
   • Mac と iPad は おなじ Wi-Fi（ゲストWi-Fiだと だめなことが多い）
   • Mac の ターミナルで python3 serve_lan.py を 動かしたまま
   • いちばんかんたん: ターミナルに でる …/ipad-check.html を いちばんさきに Safari でひらく
     → 「つながってる」とでたら LAN OK。つぎに ほんばんの http://…:8765/ をひらく
   • ターミナルに でる http://192.168.数字.数字:8765/ を まるごとコピー（x は そのままにしない）
   • https ではなく http
   • iPad の 127.0.0.1 は Mac ではない（iPad 自身を指す）。必ず Mac の IP か *.local
   • iCloud プライベートリレー ON のとき、LAN がつながらないことがある → 設定でオフを試す
   • 「読み込み中」のまま → http://…:8765/index.html?nosw=1 を試す
   • Mac の システム設定 → ネットワーク → ファイアウォール → Python を ブロックしてたら 許可

6) iPad だけひらかれへん（LANがじゃま）→ トンネル（いちばん確実）
   Mac で:
     brew install cloudflared
     ./ipad-tunnel.sh
   と表示された https://〜〜.trycloudflare.com を iPad に はる

7) ずっと同じURLがほしい
   https://app.netlify.com/drop に このフォルダをドロップ → https://〜.netlify.app
